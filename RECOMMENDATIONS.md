# คำแนะนำการเพิ่ม Interface สำหรับหน้าเว็บคู่มือส่งตรวจ

เว็บนี้ใช้สำหรับให้ลูกค้าเข้าดาวน์โหลด **PDF คู่มือการส่งตรวจวิเคราะห์** (NK Test / Masuyama's Test)  
ด้านล่างเป็นแนวทางว่าควรเพิ่มส่วนไหนบ้างเพื่อให้ลูกค้าใช้สะดวกและลดคำถามซ้ำ

---

## 1. แนะนำให้เพิ่ม (ความสำคัญสูง)

### 1.1 ขั้นตอนการส่งตัวอย่าง (Steps / Flow)
- **ทำไม:** ลูกค้ามักสงสัยว่าส่งอย่างไร กี่ขั้นตอน ใช้เวลากี่วัน
- **รูปแบบ:** บล็อกสั้นๆ แบบ STEP 1 → 2 → 3 (เช่น รับคู่มือ → เตรียมตัวอย่าง → ส่ง/นัดรับ → รอผล)
- **ตัวอย่างข้อความ:**  
  - STEP 1: ดาวน์โหลดคู่มือและตรวจสอบรายการที่ต้องเตรียม  
  - STEP 2: เตรียมตัวอย่างและเอกสารตามคู่มือ  
  - STEP 3: ส่งตัวอย่างที่จุดรับหรือนัดรับจากทีม MEDEZE  
  - STEP 4: รับผลวิเคราะห์ตามช่องทางที่แจ้งไว้  

### 1.2 ปุ่ม/ลิงก์ “คำถามที่พบบ่อย” (FAQ)
- **ทำไม:** ลดโทร/ไลน์ถามซ้ำ เช่น ส่งที่ไหน เก็บตัวอย่างอย่างไร ใช้เวลากี่วัน
- **รูปแบบ:** ลิงก์ไปหน้าหรือส่วน FAQ ในหน้าเดียวกัน (accordion หรือแยกหน้า)
- **หัวข้อตัวอย่าง:**  
  - ส่งตัวอย่างที่ไหน / รับไปเก็บที่บ้านได้ไหม  
  - ควรอดอาหาร/ยาอะไรก่อนเก็บเลือดไหม  
  - ใช้เวลาวิเคราะห์กี่วัน รับผลทางไหน  
  - กรณีตัวอย่างเสียหรือส่งผิดต้องทำอย่างไร  

### 1.3 ช่องทางติดต่อเพิ่ม (ไลน์ / แชท)
- **ทำไม:** ลูกค้าชอบสอบถามผ่านไลน์หรือแชทมากกว่าคุยโทร
- **รูปแบบ:** ปุ่ม “สอบถามผ่าน LINE” หรือ “ติดต่อทีม NK” ที่ชัดเจน (อาจอยู่ติดกับบล็อก “ติดต่อเรา” ที่มีอยู่แล้ว)

---

## 2. พิจารณาเพิ่ม (ความสำคัญปานกลาง)

### 2.1 ลิงก์ไปหน้าบริการที่เกี่ยวข้องบนเว็บหลัก
- **ตัวอย่าง:** ลิงก์ไป [MEDEZE NK](https://www.medezegroup.com/) บริการ NK Test / Masuyama เพื่อให้ลูกค้าอ่านรายละเอียดบริการก่อนส่งตัวอย่าง
- **รูปแบบ:** ปุ่มหรือข้อความใต้การ์ด PDF เช่น “ดูรายละเอียดบริการ NK Test”

### 2.2 วันที่อัปเดตคู่มือ (Last updated)
- **ทำไม:** ให้ลูกค้ามั่นใจว่าคู่มือเป็นเวอร์ชันล่าสุด
- **รูปแบบ:** ข้อความเล็กใต้ปุ่มแต่ละ PDF เช่น “อัปเดต: มี.ค. 2568”

### 2.3 ปุ่ม “ดาวน์โหลดคู่มือทั้งหมด” (ZIP)
- **ทำไม:** ถ้ามีหลายไฟล์ (เช่น หลายภาษา หรือ NK + Masuyama) ลูกค้าอาจอยากโหลดครั้งเดียว
- **รูปแบบ:** ปุ่มเสริม “ดาวน์โหลดคู่มือทั้งหมด (ZIP)” ถ้ามีไฟล์รวมให้โหลด

---

## 3. ถ้ามีระบบหลังบ้าน (ความสำคัญตามความพร้อม)

### 3.1 ใส่เลขที่อ้างอิง / Code ลูกค้า
- **ทำไม:** ถ้าลูกค้ามีเลขที่อ้างอิงหรือรหัสจากทางบริษัท สามารถใส่ในหน้าคู่มือเพื่อให้ทีมติดตามได้
- **รูปแบบ:** ช่องกรอก “เลขที่อ้างอิง” (ไม่บังคับ) หรือลิงก์ไปฟอร์มลงทะเบียนส่งตรวจ

### 3.2 ตารางนัดรับ / จุดส่งตัวอย่าง
- **ทำไม:** ลูกค้าเห็นว่าวันไหนส่งได้ ที่ไหนบ้าง
- **รูปแบบ:** ตารางหรือปฏิทินแบบง่าย หรือลิงก์ไปหน้านัดรับบนเว็บหลัก/ระบบนัดหมาย

---
